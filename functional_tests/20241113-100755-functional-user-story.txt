Functional Test Cases:

1. Credit Card Due Reminder:
   - Verify if an automated notification is sent to the cardholder reminding them of their upcoming credit card payment due date.
     - Input: Due date is upcoming
     - Expected Output: Automated notification sent to cardholder
   - Verify if the notification includes the last 4 digits of the credit card number for identification purposes.
     - Input: Credit card number
     - Expected Output: Last 4 digits of the card number included in the notification

2. Overdue Balance Alert:
   - Verify if an alert is sent to the cardholder if they miss their payment due date.
     - Input: Missed payment due date
     - Expected Output: Alert sent to cardholder
   - Verify if the alert includes the last 4 digits of the credit card number.
     - Input: Credit card number
     - Expected Output: Last 4 digits of the card number included in the alert

3. Collection Notification:
   - Verify if a formal collection notification is sent to the cardholder when an account becomes significantly delinquent.
     - Input: Significantly delinquent account
     - Expected Output: Collection notification sent to cardholder
   - Verify if the notification includes the last 4 digits of the credit card number.
     - Input: Credit card number
     - Expected Output: Last 4 digits of the card number included in the notification

4. Payment Plan Proposal:
   - Verify if a payment plan proposal is offered to the cardholder when they are unable to pay the full overdue balance at once.
     - Input: Inability to pay full overdue balance
     - Expected Output: Payment plan proposal offered to cardholder
   - Verify if the payment plan proposal includes the last 4 digits of the credit card number.
     - Input: Credit card number
     - Expected Output: Last 4 digits of the card number included in the proposal

5. Collection Agency Involvement:
   - Verify if a collection agency is involved if the cardholder fails to respond to previous notifications and reminders.
     - Input: Failure to respond to notifications and reminders
     - Expected Output: Collection agency is involved
   - Verify if the collection agency is provided with the last 4 digits of the credit card number for identification purposes.
     - Input: Credit card number
     - Expected Output: Last 4 digits of the card number provided to the collection agency

6. Legal Action Initiation:
   - Verify if legal action is initiated against the cardholder in extreme cases of non-payment or default.
     - Input: Extreme cases of non-payment or default
     - Expected Output: Legal action initiated
   - Verify if any legal documentation includes the last 4 digits of the credit card number for identification purposes.
     - Input: Credit card number
     - Expected Output: Last 4 digits of the card number included in the legal documentation

Non-Functional Test Cases:

1. Security:
   - Verify if only the last 4 digits of the credit card number are included in any communication or documentation related to the collection process.
     - Input: Communication or documentation related to collection process
     - Expected Output: Only the last 4 digits of the card number included
   - Verify if the full credit card number is never shared or displayed in plain text.
     - Input: Full credit card number
     - Expected Output: Full card number not shared or displayed in plain text

2. Performance:
   - Verify the system's performance under peak load conditions, such as when multiple notifications are sent at the same time.
   - Verify the system's response time for sending notifications and alerts.

3. Usability:
   - Verify if the notifications and alerts are user-friendly and easy to understand.
   - Verify if the notifications and alerts are sent in a timely manner.

4. Reliability:
   - Verify if the system can recover from failures, such as network issues, and still send the notifications and alerts correctly.
   - Verify if the system maintains a log of all sent notifications and alerts for auditing purposes.
